# 보안 정책

## 지원되는 버전

현재 보안 업데이트를 받는 버전:

| 버전 | 지원 여부 |
| --- | --- |
| 0.2.x | :white_check_mark: |
| < 0.2 | :x: |

## 보안 취약점 보고

보안 취약점을 발견하셨다면 **공개 이슈로 보고하지 마세요**.

대신 다음 방법으로 보고해주세요:

1. **이메일**: sales@com.dooray.com
2. 제목에 `[SECURITY]`를 포함해주세요
3. 다음 정보를 포함해주세요:
   - 취약점 설명
   - 재현 방법
   - 영향 범위
   - 가능한 해결 방법

## 보안 모범 사례

프로젝트를 사용할 때 다음 보안 모범 사례를 따라주세요:

### 환경 변수
- `.env.local` 파일을 절대 커밋하지 마세요
- API 키와 시크릿은 환경 변수로 관리하세요
- 프로덕션 환경에서는 강력한 시크릿을 사용하세요

### 데이터베이스
- Row Level Security (RLS) 정책을 활성화하세요
- 정기적으로 백업을 수행하세요
- 최소 권한 원칙을 따르세요

### API
- Rate limiting을 구현하세요
- 모든 입력을 검증하세요
- CORS 설정을 엄격하게 관리하세요

### 의존성
- 정기적으로 `npm audit`를 실행하세요
- 의존성을 최신 상태로 유지하세요
- Dependabot을 활성화하세요

## 응답 시간

보안 취약점 보고에 대한 응답 시간:

- 초기 응답: 48시간 이내
- 취약점 확인: 7일 이내
- 패치 배포: 심각도에 따라 14-30일

## 공개 정책

취약점이 수정되면:

1. 패치가 포함된 새 버전을 릴리스합니다
2. CHANGELOG.md에 보안 수정 사항을 기록합니다
3. 보고자에게 크레딧을 제공합니다 (원하는 경우)

감사합니다!
